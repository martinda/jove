Building Jove from Source

There are two general approaches to building Jove from the source code. The first is using the supplied build.xml to compile it using Ant. The other involves importing the projects into the Eclipse IDE and building it there. For both methods you'll need to do the following:

  • Download the jove-src-1.0.zip file and unzip it into a directory.
  • Download and install Apache Ant (the Jove build has been tested with version 1.6.5).
  • Download and install JavaCC (the Jove build requires at least 4.0beta1).
  • Set the JAVACC_HOME environment variable to the root directory of your javacc installation (i.e. ${JAVACC_HOME}/bin/javacc should exist).

Compiling using Ant

To compile the source code with Ant, go to the directory where you unzipped the Jove source code and enter the directory named java. Typing the command "ant" will compile the Jove source code.

Compiling using Eclipse

To compile the source code with Eclipse, you will need a version of Eclipse that supports Java 5.0. The recommended Eclipse version is 3.1. Some newer 3.2 milestones contain bugs that prevent Jove from compiling. Once you've installed Eclipse, run it and perform the following tasks:

  • Go to Window->Preferences->Java->Build Path->Classpath Variables and add a variable named ANT_HOME. This variable should point to the root of your ant installation (i.e. ${ANT_HOME}/bin/ant should exist).
  • Go to Window->Preferences->Java->Compiler and change the Compiler compliance level to 5.0.
  • Choose File->Import and select Existing Projects into Workspace. Click "Next".
    • In the "Select root directory" field, enter the directory into which you unzipped the Jove source distribution.
      • Check all the available projects (there should be 9) and click "Finish".
  • The projects will try to build as they are imported. Due to the import order, this will result in many of the projects containing errors. To correct this, select Project->Clean->Clean all projects and click "OK". Eclipse will clean all projects in the workspace and rebuild them. After the rebuild is complete, there should be no errors in the Jove projects.
